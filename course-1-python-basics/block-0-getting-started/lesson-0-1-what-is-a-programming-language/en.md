## Explanation

Before you install anything or type a single line of code, let's answer the big question: **what is a programming language, and why do we need one?**

A computer is amazingly fast but not clever on its own. It only follows **instructions** — very exact, step-by-step instructions. The catch: computers don't understand English or Russian. They need instructions written in a special language they can follow.

A **programming language** is how *you* (a human) write instructions that a *computer* can follow. **Python** is one of those languages — and it's one of the friendliest to start with. That's the language you'll learn in this course.

Think of it like a **recipe**: you (the cook) write clear steps, the **kitchen** (the computer) follows them, and out comes a **dish** (the result on screen). A programming language is the way you write the recipe so the kitchen understands it.

### Step 1: Languages are everywhere

People use languages to share ideas: English, Russian, sign language. Computers need a language too — but a much simpler, stricter one. There is no "you know what I mean" for a computer. Every step must be spelled out.

### Step 2: What is a program?

A **program** is just a list of instructions, run in order, top to bottom. That's it. A game, a website, and a calculator are all programs — long lists of small, exact steps.

### Step 3: Why Python?

- It reads almost like English: `print("Hello!")` does what it looks like.
- It's used by real creators — games, websites, science, art.
- It's forgiving for beginners and shows results fast.

### Step 4: What does code actually look like?

Here are two real lines of Python. You don't need to run them yet — just notice how readable they are:

```python
print("Hello, creator!")
print("Let's learn to code.")
```

`print(...)` means "show this text on the screen." The words inside the quotes are exactly what appears.

## Quick Drills

No computer needed — these are thinking drills:

1. **Robot sandwich.** Write 3–5 instructions to make a sandwich *as if telling a robot that takes you literally*. What happens if you skip "open the jar"?
2. **Spot the program.** Name three things around you that are run by programs (phone, microwave, game console…).
3. **Order matters.** Put these in the right order: *pour milk*, *get a glass*, *drink*. Why does order matter to a computer?

## Practice Task

**Quest name:** Be the Programmer

On paper or in a notes app, write a short "program" in plain English: **step-by-step instructions to draw a smiley face**. Use one instruction per line, in order.

**Rules of the game:**
- Each line is one tiny step (e.g. "draw a big circle").
- No step may say "you know what I mean" — be exact.
- Bonus: give your program a name, like `draw_smiley`.

There's no wrong answer — the goal is to *think like a programmer*: small, exact, ordered steps.

---

## Debug Corner

**Problem:** You tell the robot: *"Make tea."* It just stands there.

**Cause:** "Make tea" is not exact enough for a computer. It needs every step: fill the kettle, turn it on, wait, pour, add the tea bag…

**Fix:** Break big ideas into small, exact steps. This is the #1 skill in programming — computers do *exactly* what you say, not what you mean.
