## Объяснение

Заклинание `print()` ты уже знаешь с Уровня 1. Ходить по папкам — с Уровня 2. Теперь освоишь **последовательность запуска** — точные шаги, чтобы скрипт работал **каждый раз**.

**Что уже было на Уровне 1:** создать `hello.py`, один `cd`, один запуск.

**Что нового на Уровне 3:** правильное сохранение, запуск из **нужной папки**, исправление `can't open file`.

**Уже делал `my_intro.py` на Уровне 1?** Отлично — здесь главное **последовательность запуска**, а не новый синтаксис.

Последовательность запуска:

1. **Создать** файл `.py` в VS Code
2. **Сохранить** (расширение `.py`, не `.py.txt`)
3. **cd** в папку с файлом
4. **python файл.py** — старт!

---

### Шаг 1: Файлы `.py`

Файл с окончанием `.py` — скрипт Python. Windows иногда скрывает расширения — файл может быть `hello.py.txt`.

**В VS Code:** на вкладке должно быть `hello.py`, не `hello.py.txt`.

**В проводнике:** **Вид → Показать → Расширения имён файлов**.

---

### Шаг 2: Сохранить и Сохранить как

- **Сохранить (Ctrl+S)** — обновить открытый файл.
- **Сохранить как** — **новый** файл с новым именем. Так создашь `launch.py`.

**Правило:** после каждого изменения — Сохранить!

---

### Шаг 3: Одна папка — два вида

1. **Проводник VS Code** (слева)
2. **Терминал** — путь в приглашении должен совпадать

Если в проводнике видишь `hello.py`, а терминал в другой папке — `python hello.py` не сработает.

---

### Шаг 4: Открой скрипт урока

[starter/hello.py](starter/hello.py) — то же заклинание с Уровня 1, для повторения.

### Шаг 5: cd в папку урока

### Шаг 6: Проверка dir

Из папки урока:

```text
dir starter
```

**Ожидается:** в списке есть `hello.py`.

И `hello.py`, и твой будущий `launch.py` лежат в **`starter/`** — в одной папке.

### Шаг 7: Запуск из терминала

```text
python starter\hello.py
```

**Mac/Linux:** используй прямые слэши — `python starter/hello.py`. Список файлов — `ls` вместо `dir`.

**Ожидаемый вывод:**

```text
Hello, World!
I am learning Python!
My code is alive!
```

**Терминал:** `python starter\hello.py` — главный способ.

**Кнопка Run** (треугольник справа вверху) — тоже работает. Смотри вывод в панели терминала.

## Задание для практики

**Квест:** Двойной запуск

1. Открой папку **`starter/`** в VS Code.
2. Создай файл **`starter/launch.py`** (рядом с `hello.py`).
3. Две строки `print()`:
   - Приветствие
   - Любимое хобби
4. **Сохрани** — имя `launch.py`, не `.txt`
5. В терминале `cd` в **папку урока** (родитель `starter/`, не внутрь `starter/`, если не Путь B).
6. Запусти по очереди:

```text
python starter\hello.py
python starter\launch.py
```

**Путь B — из папки `starter/`:**

```text
cd starter
python hello.py
python launch.py
```

7. Должен быть вывод **обоих** скриптов (пять строк всего).

**Бонус:**

- Сохрани как `launch.py.txt` специально — почини, пока не заработает.
- **Copy path** у папки в VS Code — сравни с терминалом.

## Уголок отладки

**Проблема:** `python: can't open file 'launch.py': [Errno 2] No such file or directory`

**Причины:**

1. Терминал в **не той папке**
2. **Опечатка** в имени файла
3. Файл сохранён как **`launch.py.txt`**

**Решение:**

1. `dir` — есть ли файл? Как точно называется?
2. `cd` в папку с файлом.
3. Включи расширения в проводнике, переименуй в `.py`.
4. Запусти с точным именем из `dir`.
