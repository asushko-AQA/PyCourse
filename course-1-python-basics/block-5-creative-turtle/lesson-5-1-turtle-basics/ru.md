## Объяснение

Добро пожаловать в **Студию Turtle**! В Python есть рисовальной робот **turtle**. Ты даёшь команды — вперёд, поворот налево — и он оставляет след на экране, как ручка на бумаге.

| Команда | Что делает |
|---------|------------|
| `import turtle` | Подключает набор для рисования (stdlib — без pip!) |
| `t = turtle.Turtle()` | Создаёт черепашку с именем `t` |
| `t.forward(100)` | Движение на 100 пикселей вперёд с линией |
| `t.left(90)` | Поворот налево на 90 градусов |
| `turtle.done()` | Окно остаётся открытым, пока ты его не закроешь |

Ты уже знаешь **циклы** из Блока 3. Вместо четырёх одинаковых строк для квадрата — цикл:

```python
for side in range(4):
    t.forward(100)
    t.left(90)
```

Для треугольника нужно **три** стороны и поворот **120°** (360 ÷ 3 = 120).

## Пример кода

**Файл: [starter/shapes.py](starter/shapes.py)**

```python
import turtle

t = turtle.Turtle()
t.speed(3)

for side in range(4):
    t.forward(100)
    t.left(90)

t.penup()
t.forward(150)
t.pendown()

for side in range(3):
    t.forward(100)
    t.left(120)

turtle.done()
```

- `penup()` / `pendown()` поднимают «ручку», чтобы черепашка двигалась без линии между фигурами.

## Уголок отладки

**Проблема:** Окно turtle мелькает и сразу закрывается — почти ничего не видно.

**Причина:** Нет `turtle.done()` (или строка закомментирована). Без неё Python завершается и закрывает окно.

**Исправление:** Добавь **последней строкой** скрипта:

```python
turtle.done()
```

Сохрани, запусти снова и закрой окно, когда закончишь.
