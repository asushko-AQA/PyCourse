## Объяснение

В [Уроке 0.2](../../block-0-environment-setup/lesson-0-2-how-the-web-works/README.md) ты узнал общую схему: **браузер** запрашивает страницу, **сервер** отправляет **ответ**. Сегодня сервер — это ты: Flask слушает запросы, а твой Python-код решает, что отправить обратно.

**Flask** — небольшая библиотека Python, которая связывает URL с функциями. Одна функция отвечает за один адрес — как домофон с особым сообщением.

### Шаг 1: Активируй venv и открой стартовый файл

1. Открой терминал VS Code.
2. Активируй `.venv` (см. [Урок 1.1](../lesson-1-1-installing-flask/README.md), если забыл команду).
3. Открой [starter/app.py](starter/app.py).

Файл уже создаёт `app = Flask(__name__)`. Тебе нужно добавить маршрут и строку `app.run()`.

---

### Шаг 2: Что такое маршрут?

**Маршрут** (route) связывает **путь URL** (например `/`) с **функцией Python**.

- `@app.route("/")` — при заходе на главный адрес вызвать эту функцию
- `def home():` — функция, которая выполняется
- `return "Hello, Web World!"` — текст, который уходит в браузер

Заполни TODO так:

```python
@app.route("/")
def home():
    return "Hello, Web World!"


if __name__ == "__main__":
    app.run(port=5000, debug=True)
```

**`debug=True`** помогает при обучении — Flask перезагружается при сохранении файла. На реальном интернет-сайте его выключают (мы пока работаем локально).

### Шаг 4: Запусти сервер

```text
python starter\app.py
```

**Ожидаемый вывод в терминале (похожий):**

```text
 * Serving Flask app 'app'
 * Debug mode: on
 * Running on http://127.0.0.1:5000
Press CTRL+C to quit
```

Терминал выглядит **занятым** — это нормально. Сервер ждёт посетителей.

**`127.0.0.1`** означает «этот компьютер». **`5000`** — **порт**, как номер двери на твоей машине.

---

### Шаг 5: Открой страницу в браузере

1. Открой Chrome, Edge или Firefox.
2. В адресной строке введи:

```text
http://127.0.0.1:5000
```

3. Нажми **Enter**.

**Ожидаемый текст в браузере:**

```text
Hello, Web World!
```

Ты только что отдал веб-страницу из Python. Уровень пройден!

---

### Шаг 6: Останови сервер

Щёлкни по панели терминала, нажми **Ctrl+C**.

Сервер остановится, и появится обычная строка ввода. Всегда останавливай сервер, когда закончил — иначе порт останется занятым.

---

### Шаг 7: Измени сообщение (проверка)

1. Измени строку `return`, например: `return "Hi from my Flask app!"`
2. Сохрани файл.
3. Снова выполни `python starter\app.py`.
4. Обнови страницу в браузере — видишь новый текст?

При `debug=True` Flask иногда перезагружается сам. Если изменений нет — **Ctrl+C** и запуск заново.

## Уголок отладки

**Проблема:** Браузер пишет «Unable to connect» или «refused to connect»

**Причина:** Сервер Flask не запущен или терминал закрыт.

**Решение:** Снова выполни `python starter\app.py`. Держи терминал открытым, пока смотришь сайт.

---

**Проблема:** `Address already in use` — порт 5000 занят

**Причина:** Старый сервер ещё работает в другом терминале.

**Решение:** Найди тот терминал и нажми **Ctrl+C**, или закрой его. Затем запусти снова.

---

**Проблема:** После правок показывается старый текст

**Причина:** Кэш браузера или сервер не перезагрузился.

**Решение:** Жёсткое обновление (**Ctrl+F5**) или остановка и новый запуск сервера.
